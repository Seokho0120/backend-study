# 배포

## 배포란?

지금은 서버가 내 맥북에서만 실행됨.

```
배포 전                              배포 후
────────────────────────────────────────────────────
localhost:3000 에서만 접근 가능      https://내서비스.com 으로 누구나 접근
맥북 닫으면 서버 꺼짐               24시간 외부 서버에서 실행
다른 사람은 접근 불가               내 맥북 꺼도 서버 살아있음
```

---

## 배포 플랫폼 종류

### PaaS — 서버 관리 자동 (초보자 친화)

```
Render    → 무료 플랜 있음. 요즘 Heroku 대안으로 인기
Railway   → 월 $5 무료 크레딧. 소규모 프로젝트에 적합
Fly.io    → 무료 플랜 있지만 신용카드 필요
Heroku    → 원조. 무료 플랜 폐지 후 인기 줄어듦
```

코드만 올리면 OS, 런타임, 포트 설정 등을 플랫폼이 알아서 처리해줌.

### IaaS — 서버 직접 관리 (자유도 높음)

```
AWS EC2   → 가장 많이 쓰임. 설정 복잡하지만 자유도 높음
GCP       → 구글 클라우드
Azure     → 마이크로소프트 클라우드
```

서버 OS, Node.js 설치, 포트, 방화벽까지 전부 직접 설정해야 함.

## 전통 서버 vs 서버리스

### 전통 서버 (Render, AWS EC2 등)

```
서버 컴퓨터
┌─────────────────────────────────┐
│  Node.js 프로세스               │
│  ┌─────────────────────────┐   │
│  │  app.listen(3000)       │   │  ← 항상 켜져서 대기
│  │  Prisma DB 연결 유지     │   │
│  │  요청 오면 처리          │   │
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

카페 비유: **항상 문 열고 있는 카페**. 손님 없어도 바리스타가 자리 지킴.

### 서버리스 (Vercel, AWS Lambda 등)

```
서버리스 환경
┌─────────────────────────────────┐
│  요청 없을 때 → 아무것도 없음   │
│                                 │
│  요청 들어오면:                  │
│  ┌─────────────────────────┐   │
│  │  함수 실행 → 처리 → 종료│   │  ← 요청마다 새로 켜지고 꺼짐
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

카페 비유: **배달 전문 팝업 카페**. 주문 들어올 때만 열고, 배달 끝나면 문 닫음.

## Express + Prisma가 서버리스에 안 맞는 이유

```
전통 서버                          서버리스
────────────────────────────────   ────────────────────────────
app.listen() 한 번 실행            요청마다 함수 새로 실행
Prisma DB 연결 유지 · 재사용       요청마다 DB 연결 새로 맺음

→ 효율적                           → DB 연결 비용 매 요청마다 발생
                                     "연결 풀 초과" 오류 자주 발생
```

Prisma는 DB 연결을 유지하면서 재사용하는 방식 → 서버리스와 충돌.
Vercel은 Next.js 같은 프론트 프레임워크와 조합할 때 잘 맞음.

---

## 서버리스가 잘 맞는 경우

```
잘 맞음                            안 맞음
────────────────────────────────   ────────────────────────────
간단한 API 함수                    Express 전통 서버
Next.js API Routes                 WebSocket (실시간 연결)
이미지 변환, 이메일 발송           Prisma + DB 연결 유지 필요
트래픽이 불규칙한 서비스
```

## 우리 프로젝트 배포 계획

Express + Prisma + PostgreSQL 조합 → 전통 서버 방식 필요.

```
추천: Render (서버, 무료) + Neon (PostgreSQL, 무료)
```

- Render → Node.js 서버 배포, 무료 플랜, GitHub 연동 자동 배포
- Neon → 서버리스 PostgreSQL, 무료 플랜 (0.5GB)
- 두 서비스 모두 회원가입만 하면 완전 무료
