# 서버 기초

## 백엔드가 뭔가?

인스타그램을 예로 들면. 앱을 열면 피드가 뜨는데, 그 데이터는 어디서 오는 걸까?

```
내 폰 (앱)                                    인스타그램 서버
┌──────────────┐                              ┌──────────────────────┐
│              │                              │                      │
│  피드 화면   │  "피드 데이터 줘"  ────────► │  요청 받아서         │
│  로딩 중...  │                              │  DB에서 게시글 조회  │
│              │  ◄──────────────  데이터     │  → 응답              │
│  게시글 표시 │                              │                      │
└──────────────┘                              └──────────────────────┘
      프론트                                          백엔드
  (내가 보는 화면)                            (서버 + DB, 눈에 안 보임)
```

- **프론트엔드** — 사용자가 보고 조작하는 화면. 브라우저나 앱에서 실행됨
- **백엔드** — 서버 + DB. 화면은 없고 데이터를 처리하고 저장하는 역할

## 서버란?

**항상 켜져 있는 컴퓨터 + 거기서 돌아가는 프로그램.**

```
일반 컴퓨터                    서버용 컴퓨터
┌─────────────┐               ┌─────────────┐
│ 내가 쓸 때만│               │ 24시간      │
│ 켜져 있음   │               │ 365일 켜있음│
│             │               │             │
│ 유튜브 봄   │               │ 요청 오면   │
│ 게임함      │               │ 바로 처리   │
└─────────────┘               └─────────────┘
```

카페 비유:
```
손님 (프론트)   →   "아메리카노 주세요" (요청)   →   바리스타 (서버)
손님 (프론트)   ←   아메리카노 (응답)            ←   바리스타 (서버)
```

## Node.js란?

원래 JS는 브라우저에서만 실행됐음. Node.js는 JS를 **브라우저 밖에서도** 실행할 수 있게 해주는 환경.

```
Node.js 등장 전                Node.js 등장 후
┌──────────────────┐           ┌──────────────────────────┐
│                  │           │                          │
│  JS = 브라우저   │           │  JS = 브라우저 + 서버    │
│       전용       │           │       + 파일 읽기        │
│                  │           │       + DB 연결 ...      │
└──────────────────┘           └──────────────────────────┘
```

브라우저 JS vs Node.js 차이:

```
브라우저 JS               Node.js
┌─────────────┐           ┌─────────────┐
│ DOM 조작    │           │ 파일 읽기   │
│ fetch()     │           │ 서버 열기   │
│ 화면 이벤트 │           │ DB 연결     │
│             │           │ OS 명령 실행│
└─────────────┘           └─────────────┘
  화면 조작용               서버 개발용
```

## Express란?

Node.js로 서버를 만들 때 도와주는 **프레임워크**.

Node.js만 쓰면 URL 분기, body 파싱, 에러 처리 등을 전부 직접 구현해야 해서 코드가 복잡해짐. Express가 이걸 편하게 해줌.

```js
// Express 기본 구조
const app = express()

app.get('/posts', (req, res) => {   // GET /posts 요청이 오면
  res.json({ posts: [] })           // 이렇게 응답해
})

app.listen(3000)                    // 3000번 포트에서 대기 시작
```

## 라우팅

URL + HTTP 메서드 조합으로 어떤 코드를 실행할지 결정하는 것.

```
GET    /posts          → 목록 조회
GET    /posts/:id      → 단건 조회
POST   /posts          → 생성
PATCH  /posts/:id      → 수정
DELETE /posts/:id      → 삭제
```

URL은 명사(리소스), 행위는 HTTP 메서드로 표현하는 게 REST 원칙.

---

## req / res

```
req (request)  → 클라이언트가 보낸 것
  req.body        → POST/PATCH 요청의 body (fetch 시 넘긴 JSON)
  req.params.id   → URL의 :id 값 (/posts/5 → "5")
  req.query       → 쿼리스트링 (/posts?page=1 → { page: "1" })
  req.headers     → Authorization 등 헤더

res (response) → 서버가 돌려줄 것
  res.json(data)         → JSON 응답
  res.status(404).json() → 상태코드 + JSON 응답
```

## 미들웨어

요청이 라우터에 도달하기 전에 실행되는 함수.
프론트의 **axios interceptor**랑 비슷한 개념.

```
요청 들어옴
    │
    ▼
express.json()    ← body를 JSON으로 파싱
    │
    ▼
authMiddleware    ← 토큰 검증 (인증 필요한 라우트만)
    │
    ▼
라우터 (실제 코드 실행)
    │
    ▼
응답
```

```js
app.use(express.json())           // 이거 없으면 req.body가 undefined
app.use(express.static('public')) // 정적 파일 서빙

// 특정 라우트에만 적용
router.post('/', authMiddleware, (req, res) => { ... })
```

순서가 중요함. 위에서 아래로 순서대로 실행됨.

## 에러 처리

각 라우터에서 try-catch로 에러를 잡아 next(err)로 전달.
index.js 맨 아래에 전역 에러 핸들러 등록. 인자가 반드시 4개여야 Express가 에러 핸들러로 인식함.

```js
// 라우터
try {
  ...
} catch (err) {
  next(err)
}

// index.js 맨 아래
app.use((err, req, res, next) => {
  res.status(err.status ?? 500).json({ message: err.message })
})
```

## HTTP 상태코드

| 코드 | 의미 |
|------|------|
| 200 | 성공 |
| 201 | 생성 성공 |
| 400 | 잘못된 요청 (클라이언트 실수) |
| 401 | 인증 필요 |
| 403 | 권한 없음 (인증은 됐지만 접근 불가) |
| 404 | 리소스 없음 |
| 500 | 서버 내부 오류 |

## 전체 흐름 (한눈에)

```
사용자 행동                 일어나는 일
─────────────────────────────────────────────────────────

① 브라우저에서             프론트 JS 실행
  게시글 목록 접속          fetch('/posts') 호출
                                │
                                ▼
② 요청이 인터넷 타고        서버 컴퓨터에 도착
  서버로 날아감              Node.js + Express가 받음
                                │
                                ▼
③ 서버가 요청 처리          app.get('/posts', ...) 코드 실행
                              DB에 SQL 쿼리 전송
                              SELECT * FROM posts
                                │
                                ▼
④ DB가 데이터 꺼내서        서버에 결과 전달
  서버로 돌려줌              [{ id:1, title:'...' }, ...]
                                │
                                ▼
⑤ 서버가 응답 전송          res.json(posts)
  프론트로 결과 보냄         인터넷 타고 브라우저로
                                │
                                ▼
⑥ 브라우저가 받아서         화면에 게시글 목록 표시
  화면 업데이트
```
